What you get

  • Drop-in mock. Same paths, methods, status codes and content types as your API. Point tests at it; only the base URL changes.
  • Happy path by default. Every endpoint returns its success response, so existing tests pass untouched.
  • Errors on demand. Send X-Mock-Status: 404 to get any documented error response for that endpoint.
  • Kubernetes-ready. kubectl apply -f k8s/ runs the official wiremock/wiremock:3 image with your mappings as a ConfigMap.
  • Swagger 2.0? Convert it to OpenAPI 3.0 in the second tab — or just drop it in the first tab and it converts automatically.
